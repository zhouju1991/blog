Django uwsgi+nginx生产部署

我们平时在开发的时候都是使用Django自带的服务来运行的，比较不稳定，不能再生产环境上使用。本文讲解如何利用uwsgi和nginx来将Django应用部署到生产环境。

版本说明

本文使用的相关软件版本如下

Python: 3.7.9版本

Django: 3.1.2版本

nginx: 1.16.1

sqlite3: 3.33.0

相关概念说明

安装gcc套件

yum install -y gcc gcc-c++ autoconf automake

Linux下卸载Python3

rpm -qa|grep python3|xargs rpm -ev –allmatches –nodeps

whereis python3 |xargs rm -frv

查看现有Python：whereis python

Linux下Python 3 安装

安装依赖：

yum -y install zlib-devel bzip2-devel openssl-devel ncurses-devel sqlite-devel readline-devel tk-devel gdbm-devel db4-devel libpcap-devel xz-devel libffi-devel

下载：

wget https://www.python.org/ftp/python/3.7.9/Python-3.7.9.tgz

创建安装目录：mkdir -p /usr/local/python3

解压：

tar xf Python-3.7.9.tgz

cd Python-3.7.9

配置：

./configure –prefix=/usr/local/python3

编译安装：

make && make install

建立链接：

ln -s /usr/local/python3/bin/python3 /usr/bin/python3

ln -s /usr/local/python3/bin/pip3 /usr/bin/pip3

vi /etc/profile，新增：

PATH=$PATH:$HOME/bin:/usr/local/python3/bin
export PATH

source /etc/profile

查看是否安装成功：

python3 -V

pip3 -V

sqlite3升级

下载sqlite3: wget https://sqlite.org/2020/sqlite-autoconf-3330000.tar.gz

解压：tar xf sqlite-autoconf-3330000.tar.gz

cd sqlite-autoconf-3330000

配置：./configure –prefix=/usr/local

安装：make && make install

查看已安装的所有sqlite3：find /usr/ -name sqlite3

老版本：/usr/bin/sqlite3 -version

新版本：/usr/local/bin/sqlite3 -version

备份旧版本：mv /usr/bin/sqlite3 /usr/bin/sqlite3_old

建立链接：ln -s /usr/local/bin/sqlite3 /usr/bin/sqlite3

全局版本：sqlite3 –version

vi ~/.bashrc 增加行：export LD_LIBRARY_PATH=”/usr/local/lib”

uwsgi安装并使用uwsgi启动应用

安装：pip3 install uwsgi

建立链接：ln -s /usr/local/python3/bin/uwsgi /usr/bin/uwsgi

使用uwsgi启动：uwsgi –http 0.0.0.0:80 –file /root/mysite/mysite/wsgi.py –static-map=/static=static

创建uwsgi启动配置文件：

cd /root/mysite

使用配置文件启动：uwsgi –ini uwsgi.ini

停止：uwsgi –stop uwsgi.pid
